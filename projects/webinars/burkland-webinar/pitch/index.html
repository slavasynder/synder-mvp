The Problem · Why Synder

Why Synder

Reconciliation is hard for a stack of reasons. Here's how Synder makes it less so.

Why reconciliation is hard
🔀
Many-to-one matching
One sale → receipt + fee + payout. By hand, at scale.
❓
Native tools work by totals
Accounting platforms tie volume to a balance. You confirm totals, not which rows actually paired.
📑
Source is usually an Excel
For non-integrated accounts — statements, exports — shaped and compared by hand.
🧩
Hard to scale across platforms
Stripe isn't Shopify isn't PayPal. A different recipe per integration — painful across a portfolio.
🕵
Errors hide until audit
Small mismatches surface months later — by then the trail is cold.
⏳
Hours per client per month
Manual work scales with volume — and never shrinks.
What Synder brings
🔍
Auditable

See which row matched which row, side by side. Native QBO ties by totals — you can confirm numbers, not pairs.

⚙️
Automated

Synder matches and surgically points out what's wrong — which row, which side, which delta. Every fix is anchored in a transaction, not a guess.

⚡
Fast

Hours become minutes. No waiting for cash to land in the bank — platform data is available immediately.

Transaction Reconciliation

Resolution Paths Today

Three workflow actions on the rows themselves, three hygiene checks before you run.

When the result needs action
🛠
Adjust in books → refresh
Fix the discrepancy in QBO/Xero, then refresh the reconciliation. In-Synder fix is on the roadmap.
🔗
Manual matching
Pair missing rows on either side when ID-matching can't link them. Different conventions, manual entries, re-syncs.
⏭
Ignore
For rows that genuinely shouldn't be matched. Reconciliation math closes around them.
Hygiene checks before you run
🏦
Clearing accounts
Sales, refunds, fees, payouts all land in the same clearing account per integration.
🔄
Sync hygiene
No Ready-to-Sync rows. No failed or cancelled rows. No deleted transactions left unresolved.
⚡
Auto-import & Auto-sync on
Both need to be enabled. Auto-import pulls transactions into Synder; auto-sync pushes them through to accounting. Either off → Missing-in-Accounting gaps at reconciliation time.
Clean upstream = clean reconciliation. The hygiene items below the line catch the bulk of "Missing in Accounting" cases before they ever surface.

Demo of Prototypes

Six prototypes from our design team. We'll walk each one live — react to the idea, the workflow, the shape.

JE-from-Synder
Create the accounting entry for a Missing-in-Accounting row from inside Synder — no round-trip to QBO.
Balance Reconciliation
A balance-level lens inside Transaction Reconciliation. Aggregate first, then drill into rows when they disagree.
Match with AI
When ID matching fails, AI proposes a candidate based on amount, date, customer, memo — you accept or reject.
Firm-Level Reconciliation
One view across your portfolio of clients. Status, ownership, deadlines, bulk run.
Matching Rules
Configurable criteria stack (ID, amount, date, customer, memo) with tolerances. Saved presets, per-integration.
Audit Report
A final-stage shareable artifact from the reconciliation — ready to hand to an auditor, a client, or a partner.
Prototypes — not final designs. React to the idea, the workflow, the shape. Vote at the end on which ones we should ship sooner. Links coming in the follow-up email.
