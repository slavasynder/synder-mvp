Financials — Business Insights 2.0

Old standalone Business Insights, re-launched as a first-class Financials section inside Synder. Reports, P&L, Balance Sheet — all in one place.

What's there
New sidebar: Overview · My reports · Templates · P&L · Balance Sheet. Three ways to create a report: From Template · From Scratch · With AI.
Gating
Balance Sheet + P&L visible only for Per Transaction + QuickBooks Online orgs. Everyone else sees Financials without those two.
Raw edges — be honest
SET-592 — chart broken when grouped by Product · SET-597 — group-limit ignored in preview. Both shipped as "not in this quarter". Late-arriving polish bugs (SD-17759, SD-17760) fixed same-day.
The ask
Walk one customer through Financials this month. Log what they say. Adoption today: 48 users first 7d vs 300/mo target · 13 reports created in the first 30 days.

Sync Speed dashboard — Per Transaction

Customers can finally see how much they're syncing and what's slowing it down — no Support ticket required. PT orgs only (not QBD, not Summary Sync).

Graph 1 — Volume
Transactions synced per platform, aggregated daily · weekly · monthly · 6mo · 12mo · custom. Filter by platform and status (Failed / Synced / Warnings / Skipped / Canceled).
Graph 2 — Time breakdown
Avg per-transaction sync time, split by process step: integration fetch, QBO customer lookup, RevRec lookup, JE creation… Toggle to drill from group to step level.
Why this matters
Answers "how long will it take to sync my 50k historical transactions?" without a Support ticket. Gives CFTs a data-backed reason to recommend Premium dedicated when a step is a bottleneck.
Trust signal
Tooltip discloses "~8% of sync time is untracked" and "queue wait time excluded" — we're upfront about what the number means.

Automated Transaction Reconciliation — Shopify & Amazon

TxnRecon used to cover Stripe + PayPal only. Now Shopify and Amazon are first-class — no file uploads, no column mapping.

Shopify 2026-03-04
Auto-fetch "Transactions and associated payouts". Reconciles Shopify Payments clearing against QBO or Synder Summary datasource.
Amazon V2 2026-04-30
Auto-fetch Settlement reports. Reconciles against Synder Summary datasource. QBO-direct not yet.
The flow — before → after
Manual spreadsheet reconcile → Pick date range → Start matching → Matched · Discrepancy · Not Matched · Ignored
Coverage limits — don't oversell
Shopify: Shopify Payments clearing only (not "Other orders clearing"); multi-currency fragile.
Amazon: Summary datasource only; no upload/Standard mode (files are TSV, not CSV); 0-amount rows not auto-ignored.

Error Grouping in Sync Issues

Failed syncs now grouped by error message. Fix the root cause once — bulk-resync every transaction affected by it.

The mechanic
Account not found — 150 → Fix mapping → See related transactions → Select all → Sync
Where it lives
Dashboard (formerly Control Panel). Groups FAILED · CANCELED · ROLLBACK_FAILED · ROLLBACK_CANCELED · RULE_FAILED. PT users + admins see per-txn issues; only admins see datasource errors.
Adoption today
937 orgs had a groupable sync failure in the last 90 days. 118 opened the grouped view (~13%). Headroom to push in Support replies.
v1 limits
Groups by raw message string — variants like "total 40 vs 39" and "total 41 vs 40" are separate rows. Saved Search caps at ~500 webhooks per batch. Refresh lag ~1h. Archived txns excluded.
